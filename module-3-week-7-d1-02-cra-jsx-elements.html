Introduction

React is a front-end library used for building user interfaces or UI components. React provides us with the programming interface for organizing our code into components. Thanks to JSX we can put the markup, the logic, and the data together in the same file, called a component. In this lesson, we will explore the syntax of JSX and give you a chance to practice it by creating React components.

Before we dive into JSX we’ll first need to create a React application. There are several ways to create a new React project and each requires a certain basic setup. First, you have to set up the file structure and install the supporting packages. Second, you must configure the project so that all these tools work together. In this lecture, we’ll see how to use the Create React App generator to create a new React project.


React project setup

As React is only a library it depends on other tools that provide a development environment.

To set up a React project and the development environment from scratch you must install and configure the following:

  • Supporting tools:

    • A package manager, such as Yarn or npm.
    • A compiler, such as Babel - used to convert JSX into JavaScript.
    • A bundler, such as webpack or Parcel. Bundlers provide a dev server and live reloading during development. They also bundle and minify the code.

  • React Core packages:

    • React - the core React library allowing us to work with JSX,
    • ReactDOM - library for rendering/displaying the React application in the DOM.

But we don’t have to worry about setting all this up from scratch! Developers at Facebook created an excellent tool to help set up our react project. This means that we don’t have to go through the tedious setup and configuration.

Create React App is a tool to help you build React applications. It saves you from time-consuming setup and configuration. It provides an out-of-the-box application setup: packages, configuration, and folder/file structure. Create React App, or CRA for short, allows you to get started quickly and to easily set up a React project that follows best practices.

Thanks to create-react-app, we don’t have to do any manual setup and configuration of tools such as webpack and babel. They are already pre-configured.

You simply run one command, and create-react-app sets up the React project and the tools needed:

$ npx create-react-app name-of-your-app

Thanks to npx, there is no need to npm install the package create-react-app, the latest version will be downloaded and executed.

However, if you have installed the create-react-app (locally in your devDependencies or even globally), npx will use this version.

So if you have previously installed create-react-app globally, you may want to

$ npm uninstall -g create-react-app

to ensure that npx always uses the latest version.

Create React App is divided into two packages:

  • create-react-app which is a global command-line utility that is used to create new projects,
  • react-scripts which is a development dependency in the generated projects.

Create a new app

Let’s create a new React app:

$ npx create-react-app my-first-app

The above command will create a directory named my-first-app. Once the script is done running, you can open the project folder:

$ cd my-first-app
$ code .

Sidenote: Alternative ways to run the create-react-app are using:

  • npm: npm init react-app my-first-app
  • yarn: yarn create react-app my-first-app

Built-in scripts

Inside the newly created project, you can run some built-in commands:

npm start or yarn start

This will run the app in development mode. After running it, your app will launch on http://localhost:3000.

Best Part? The page will automatically reload if you make changes to the code. You will see the build errors and lint warnings in the console. No manual Webpack configuration is needed.

npm test or yarn test

Starts the test runner. create-react-app now has built-in tools to start testing your app while you are developing it, and for this, by default, you can use Jest. This functionality is out of scope for our course, but feel free to read more about testing here.

npm run build or yarn build

Builds the app for production to the build folder. It correctly bundles React in production mode, which means creates a bunch of static files and optimizes the build for the best performance. After successfully running this command, your app is ready to be deployed.

create-react-app is an open-source project and there are a couple of thousands of contributors who collaboratively work on it together with the core team. Maybe you could be one of them!

Run the React App

To run your app use the previously mentioned command:

$ npm start

As a first step, let’s replace the content generated by the Create React App with our own. Open the component App.js and remove the nested HTML content, leaving only the element <div className="App"> </div>.

After removing the nested content your App.js should look like this:

// App.js

import logo from './logo.svg';
import './App.css';

function App() {
  return (
    <div className="App">
      {/*  We removed all of the content generated by Create React App */}
    </div>
  );
}

export default App;

We can now start adding our content and components:

// App.js

import './App.css';
import Header from './components/Header';

function App() {
  return (
    <div className="App">
      
      <h1>Hi YOUR_NAME!</h1>
      
    </div>
  );
}

export default App;

Folder public

The public folder contains files that will be read by your browser while you’re developing the app; the most important of these files is index.html.

  • public/index.html is the only .html file for the entire React app. For this reason, we say that React is a Single-Page Application.

    Instead of having many HTML pages, the entire app content is managed and updated dynamically by JavaScript. The app and all its content are displayed on this HTML one page.


Folder src

The majority of the development work from our side as developers will be happening in the src folder.

  • src/index.js file is the one where the main render call is happening thanks to the ReactDOM.render() method. The method ReactDOM.render() injects the React application into the <div id="root"> so that the app can be rendered in the browser.

  • src/App.js is a React component called “App”. This component will be the root component to all the other components.

  • src/index.css comes with some basic styling for our app.

  • src/App.css stores styling targeting the App component specifically.

  • src/App.test.js is the first set of tests we can run against the sample App component that we start with.

  • package.json file lists all the dependencies and scripts needed to run the React app successfully.

  • The node_modules folder is generated automatically and contains all of the installed dependencies.

  • yarn.lock or package-lock.json (depending on the package manager) will have the locked list of all installed dependencies of our app.

Although we mentioned this above, it is worthy of mentioning again:

  • public/index.html is the only .html file for the entire React app.

    React app is displayed in this HTML page, more precisely in the <div id="root"></div>. The entire app content is managed dynamically by JavaScript and displayed on this one HTML page.

  • src/index.js file is considered to be the default entry point to any React app. This file contains the method ReactDOM.render() which injects the React application into the <div id="root"> so that the app can be rendered in the browser.

Rendering the React App in the DOM

As mentioned earlier, React is used to create Single-Page Applications. This means that React apps always have only one HTML file, index.html. This file serves as a container for loading the JavaScript code that the React app is made of and displaying it in the browser.


Rendering the App to the DOM

The index.html file contains only one element in the body, the <div id="root">.

The <div id="root"> is a container element into which the entire React app is rendered. The React app is injected (appended) to the DOM thanks to the ReactDOM.render() method.

You may wonder, if there is only one element <div id="root"> how do we end up seeing all the other DOM content?

The React app dynamically creates DOM elements from our JSX code.

Let’s show this through an example. In the index.js you will find the ReactDOM.render() method. Here you can see that the App component is being rendered to the DOM with the ReactDOM.render() method.

index.js
// index.js
// ... imports stay unchanged

ReactDOM.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
  document.getElementById('root')
);

The ReactDOM.render() method has the following syntax:

ReactDOM.render(whatToAppend, whereToAppend);

  • whatToAppend argument represents the JSX React content that we want to inject (append) to the DOM.

  • whereToAppend argument represents the DOM element to which we want to append the JSX React content.

If you remove the whatToAppend part and replace it with null you’ll notice the page in the browser turning blank. This is because nothing is rendered in the <div id="root"> in the DOM:

// index.js
// ... imports stay unchanged

ReactDOM.render(null, document.getElementById('root'));

So, to display any JSX React content on the page, we have to render it to the DOM using the ReactDOM.render() method. If you add the following JSX content you will see it appearing on the page:

// index.js
// ... imports stay unchanged

ReactDOM.render(
  <div>
    <h1> Hello Ironhacker! </h1>
    <p> You are rocking it! </p>
  </div>,
  document.getElementById('root')
);

Rendering the Root Component

We call App a root component as it is the first React component appended to the DOM. Any component, or any JSX code, we add to the App.js will be automatically appended to the DOM through the App.js.

Go back and update the index.js to the previous state:

// index.js
// ... imports stay unchanged

ReactDOM.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
  document.getElementById('root')
);

JSX and elements



JSX is a syntax extension to JavaScript. JSX may remind you of a template language, but it comes with the full power of JavaScript.

Browsers don’t understand JSX. The JSX code gets compiled into browser-understandable code thanks to Babel. This process happens automatically. All React components and any JSX code get converted to ES5 JavaScript.

The JSX code gets compiled into browser-understandable JavaScript thanks to Babel. You don’t need to install it or configure it manually, it comes together with Create React App. It is preconfigured and hidden so that you can focus on the code.

Let’s open the app. We will remove the code currently present inside the function App(){} and just leave the following structure:

// App.js

import React from 'react';
import logo from './logo.svg';
import './App.css';

function App() {
  return <div className="App"></div>;
}

export default App;

JSX Under the Hood

Let’s take a look at this variable declaration:

const element = <h1>Hello world!</h1>;

The above code represents a very simple JSX code. Use this code snippet and paste it to the https://babeljs.io/repl to see how Babel converts it into JavaScript. You will get the following JavaScript code:

var element = React.createElement('h1', null, 'Hello, world!');

Simply speaking, the elements you create using JSX are converted by Babel into plain JavaScript. React is a JavaScript library and actually uses the method React.createElement() to create the DOM elements.


Add the following code to the App.js:

// App.js
// ... imports stay unchanged

const heading = <h1>🥁🥁🥁 React is cool! 🎉</h1>;

function App() {
  return <div className="App">{heading}</div>;
}

export default App;

We added our super simple JSX element (heading) in the App.js and, since App component is already rendered in the DOM, the heading gets shown inside of the component.

If you haven’t run your app already, you can do it now and in the localhost:3000, this should be the expected result.



Embedding expressions in JSX

Two super important things to keep in mind when working with JSX:

  1. You can embed any valid JavaScript code into the JSX using the single curly braces: {}.
  2. You can use as many DOM elements as you wish inside JSX code in the React components. However, all of the elements in a component must be wrapped with a single enclosing HTML element, whichever you find the most suitable. This means that React components must always return content enclosed in a single HTML element.

Example: Wrong - component is returning two adjacent HTML elements h1 and h3

// Wrong!
function MyComponent() {
  return (
    <h1>Hello Ironhacker!</h1>
    <h3>Welcome To World of React</h3>
  )
}

Example: Correct - component is returning content wrapped in a single enclosing div tag!

// Correct!
function MyComponent() {
  return (
    <div>
      <h1>Hello Ironhacker!</h1>
      <h3>Welcome To World of React</h3>
    </div>
  );
}

The full power of JSX comes with the ability to embed any JavaScript valid expression to the JSX element.


Example 1 - embed variables:

// App.js
// ... imports stay unchanged

const heading = <h1>🥁🥁🥁 React is cool! 🎉</h1>;

const student = {
  firstName: 'ana',
  lastName: 'martinez'
};

function App() {
  return (
    <div className="App">
      {heading}

      <h3>
        Hi, {student.firstName} {student.lastName}!
      </h3>
    </div>
  );
}

Example 2 - embed functions (or methods):

// App.js
// ... everything that we already added stays unchanged, except function App()

const { firstName, lastName } = student;

function App() {
  return (
    <div className="App">
      {heading}

      <h3>
        {/* You can turn this to {firstName} {lastName} */}
        Hi, {student.firstName} {student.lastName}!
      </h3>

      <h4>
        In uppercase: {firstName.toUpperCase()} {lastName.toUpperCase()}.
      </h4>
    </div>
  );
}

Example 3 - embed function execution:

// App.js
// ... everything that we already added stays unchanged, except function App()

function capitalizeFirstLetter(str) {
  return str[0].toUpperCase() + str.slice(1);
}

function App() {
  return (
    <div className="App">
      {heading}

      <h3>
        {/* You can turn this to {firstName} {lastName} */}
        Hi, {student.firstName} {student.lastName}!
      </h3>

      <h4>
        In uppercase: {firstName.toUpperCase()} {lastName.toUpperCase()}.
      </h4>

      <h4>
        Capitalized:
        { capitalizeFirstLetter(firstName) } { capitalizeFirstLetter(lastName) }
      </h4>
    </div>
  );
}

Example 4: embed the attribute value

In JSX, we can embed the attribute values as well:

const theId = 'home';

const divElement = <div id={theId}>div with an id</div>;

At this point, you probably don’t see the use case, but when it comes to styling in React, you will have plenty of opportunities to use the above syntax.


Example 5 - embed static files: images

To load a local image with plain HTML, we have to set the src attribute of the <img> to the local path of the file. To load an external image in plain HTML, we set the URL of the file.

<!-- dummy example -->
<img src="./assets/some-image.jpg" alt="some image" />
<img src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/upload_129040de20781c45366fd2193fcbb5fd.png" alt="react is cool" />

In React, to load an image stored locally in the project, we must first import the file:

import ironhackLogo from './assets/ironhack-logo.png';

// ...

<img src={ironhackLogo} alt="ironhack logo" />

When it comes to using online-hosted images, the process stays the same as if we were using HTML.

<img src="https://s3-eu-west-1.amazonaws.com/ih-materials/uploads/upload_129040de20781c45366fd2193fcbb5fd.png" alt="react is cool" />

Self-closing tags

Any JSX element can be written as a self-closing tag:

<MyComponent />
// or
<MyComponent> </MyComponent>

<label />
// or
<label> </label>

Additionaly, self-closing HTML tags such as <br />, <hr />, and <img /> must always have a closing / to be valid JSX that can be transpiled.

<br />         // Correct
<hr />         // Correct
<img src="" /> // Correct


<br>          // Wrong!
<hr>          // Wrong!
<img src="">  // Wrong!

Attributes Names are camelCased

In React, compared to the HTML, all attributes with the - in their name are converted to their camelCase version. Example: background-color becomes backgroundColor.

Some of the attributes without any case get renamed to their camelCase equivalent:

  • onchange —> onChange
  • onclick —> onClick
  • onsubmit —> onSubmit

Overall exceptions: Because they are reserved words in JavaScript, class and for become:

  • class —> className

  • for —> htmlFor


Example

<label htmlFor="username" className="control-label">
  Username
</label>

Adding comments in JSX

There are two rules when it comes to commenting out the code in JSX:

  • when inside the HTML part, comments need to be in {/* some comment here */}
  • when inside JavaScript part, meaning between the curly braces where we can use the full power of JavaScript, comments are the same as in JS files // some comment here
<div>
  {/* some comment here */}

  {friends.map(elem => {
    // some comment here
    return elem.name;
  })}
</div>
