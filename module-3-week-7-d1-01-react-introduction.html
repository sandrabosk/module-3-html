Introduction

In today’s web development, there are many ways to create user interfaces. These can be simple Static Websites, Multi-page Applications (server-side rendered), Progressive Web Apps (PWA), Single-page Applications, etc.

The simplest way to create a front-end application is using plain HTML, CSS, and JavaScript. This is the purest way of doing web development. It doesn’t require the use of tools, libraries, or frameworks.

However, this approach doesn’t scale very well when creating large web applications.

As applications grow, they tend to become hard to maintain and scale. Think about a web platform such as Facebook, Airbnb, or Instagram. How many HTML, CSS, and JS files do we need to create one such application? The answer is thousands of files.

Fun fact: Facebook’s frontend runs on over 60 million lines of code . You get the picture!

As an application grows, it may start encountering architecture problems such as:

  • code and files being very hard to organize (too much code),
  • it becomes hard to interconnect or reuse logic and code across files and features,
  • It is hard to enforce specific design patterns and structure

When these things happen, a framework can come to help.


Frameworks and Code Libraries

A software framework is:

A universal, reusable software environment used to facilitate the development of software applications, products, and solutions.

A framework serves as a development environment and a set of tools and commonly includes:

  • Support programs,
  • Compilers/transpilers,
  • Code libraries (also called packages),
  • Toolsets,
  • Predefined structure of the application (folder/file structure and configuration)
  • Enforced set of rules and well-tested design principles that must be followed

A JavaScript framework is a collection of JavaScript code libraries and tools used for developing JavaScript applications.

A Framework helps you to build apps faster by addressing common development issues. You may know some other frameworks, like Express. In this module, we will use React frontend framework to create client-side applications.


Front-end Frameworks

A front-end framework consists of tools, JavaScript libraries, and packages. Front-end frameworks help developers build and scale front-end applications easier and faster.


A code library is:

A collection of objects, functions, and methods which can be reused and shared among multiple applications. A library performs specific, well-defined operations.

– Kolev (2017, July 31). _The pros and cons of using a framework

In other words, code libraries are collections of ready-to-use code used to speed up the creation of programs.

When it comes to React, there are still many divided opinions on the question, “Is React a library or is it a framework?”. React takes the best of both, and it isn’t easy to draw the line and decide in favor of one or the other.

Introduction to React

What is React?

React is a front-end JavaScript library used to create front-end applications that run in the browser. It was created by Jordan Walke, a software engineer working for Facebook. React first deployed on Facebook’s newsfeed in 2011 and on Instagram.com in 2012.

React allows developers to create robust web applications that handle dynamic changes in the data without reloading the page. Besides Web apps, we can also use React for developing Android and iOS mobile apps. This is done through the cross-platform framework called React Native.


Advantages of React

  • Component-based development - We create React applications using components. Components are smaller reusable pieces of UI. This allows us to build complex applications.

  • Separation of concerns - React enables a clean, modular design of the applications. Developers can work on different parts of the application, on a specific component. This way, they don’t interfere with the work of others or break the existing code.

  • Speed - React enables us to maintain a clear project structure. Clear structure speeds up the development.

  • Enforces Structure - React enforces certain design patterns and project structure. If all developers in the team follow the same practices, collaboration improves.


React Features

Let’s take a closer look at some essential features of React:

  • Component-Based - Components are building blocks of React applications. We can compose React components to make complex UIs. Additionally, each component can have encapsulated state.

  • JSX - JSX is a JavaScript syntax extension used to write HTML and JavaScript together. Instead of separating JavaScript and HTML into different files, React uses JSX to combine HTML and JS together. JSX speeds up and simplifies the creation of React apps and components. Don’t worry; we will take a further look into this in the next steps!

  • Learn Once, Write Anywhere - React can be used to pre-render web pages on the server with Node. Besides Web apps, we can use React to create mobile apps with React Native.

While JSX is the recommended way for creating React apps, it is not the only one. We can also create React apps using pure old JavaScript!

Add React to a Website

While React is commony used for creating Single page applications (more on that later) and with a development toolchain, this is not the only way it can be used. React can be added to an existing web page with only few lines of code, without using a toolchain. Here’s an example:

<!-- index.html -->

<!DOCTYPE html>
<html>
  <head>
    <title>Add React to a Website</title>
    <meta charset="UTF-8" />

    <link rel="stylesheet" href="./src/styles.css" />
    
    <!-- Load Babel - used to parse JSX -->
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
    <!-- React's core library-->
    <script crossorigin src="https://unpkg.com/react@17/umd/react.development.js"></script>
    <!-- ReactDOM library renders the content created with React to the DOM. -->
    <script crossorigin src="https://unpkg.com/react-dom@17/umd/react-dom.development.js"></script>
    
  </head>

  <body>
    <div id="root"></div>

    <script type="text/babel">
      ReactDOM.render(
        <div>
          <h1>Hello React!</h1>
          <p> React is a front-end JavaScript library </p>
        </div>,
        document.getElementById('root')
      )
    </script>
    
  </body>
</html>

To add React to the HTML page like in the example above, we need to do the following:

  • Create a DOM container in the HTML page <div id="root"></div>.

  • Add React Scripts.

  • Add Babel (to enable writting JSX).

The above setup is all that is needed! JSX content that is passed to the ReactDOM.render() method will be rendered to the DOM in the specified root div: document.getElementById('root').

In the above example, we rendered a fairly small amount of content using React. However, we could render as much content as needed. Let’s update the index.html page and add more content:

Component-based

Now let’s take a look at the App.js file .App.js is a React component. A React component holds UI, logic, and data, all in one place. You will notice in the example that App.js contains a code that is a mix of JavaScript and HTML, all in the same file!

You may be wondering how’s that even possible? Well, it is possible thanks to JSX. JSX allows us to put HTML, JavaScript logic, and data together in the same file. We will cover JSX in detail in a later lesson.

Component-based development brings the following benefits:

  • Code is reusable.
  • We can quickly duplicate the Components and move them around.
  • To update a part of the app, we update a single component without touching the rest of the application.

Function App is the initial component, always created first in a React app, and serves as a parent for all other content and React components.

We call App a root component, as it represents the root of the React application rendered to the DOM.

Creating a React component

You saw how components make it easy to organize the code in an application. You are now probably eager to create your first component. So, let’s create a React component.

We will continue working on the previous CodeSandbox example - React introduction. We will work on the file src/components/MyButton.js. If you open the file, you will find the following code:

// src/components/MyButton.js

function MyButton() {
  return ( /* HTML & JSX CONTENT */ );
}

export default MyButton;

Now the above function MyButton is a component. In a nutshell, a component is a function that returns JSX content. The only thing that we need to do here is add the content we want to display.

Take a moment and go over the other components inside of the /src/components folder. Take a look at the components Headline.js and Button.js. As you can see, they are very similar. Each component is a function that returns some JSX content.

A React component is just a function that returns JSX content or other components.

So, to make the above MyButton component render some content, we need to return the HTML or JSX content from the function.

Update the MyButton.js file and add the following content in the return parentheses:

// src/components/MyButton.js

<a href="https://ironhack.com">
  <button className="btn yellow">MY BUTTON</button>
</a>

After updating the file, your code should look like this:

// src/components/MyButton.js

function MyButton() {
  return (
    <a href="https://ironhack.com">
      <button className="btn yellow">REACT DOCS</button>
    </a>
  );
}

// To make the component available, we have to export it
export default MyButton;

Practice makes perfect. To enforce what you just learned, you should try to create a component from scratch. Go on and delete the existing code from the MyButton.js and recreate the component, this time from scratch.


Export a component

As you may have noticed, there is an export default statement at the bottom of the file. We need this statement to make the component available for import from other files. This means that you must include the export default statement in every component. Otherwise, the component won’t be accessible for import, and React will throw an error.

React supports the latest JavaScript syntax and uses the import and export statements for importing and exporting code from files. This syntax is also known as ES modules.

The ES modules statement:

export default ComponentName;

Is equivalent to the Node’s:

module.exports = ComponentName;

In React, we use ES modules import and export statement to import and export components.


Import and use a component

To start using the MyButton component, we need to import it into the component where we want to display it.

To do this, open the App.js file and add the following import statement at the top of the file:

// App.js

import MyButton from './components/MyButton';

Once you have imported the component, the next step is to render it inside the App. To render the component add it inside of the return as a tag, putting the angle brackets < > around the component name like: <MyButton />. Update the App component in the following way:

// App.js

function App() {
  return (
    <div className="App">
      <Navbar />
      <Headline />

      <Button />
      <Button />      

      <MyButton />

      <Gallery />
    </div>
  );
}
After updating the file, your code should look this (click to expand):
