Function and Class Components

Components let you split the UI into independent, reusable pieces, and think about each piece in isolation.

– Components and Props –. (n.d.-a). ReactJS Documentation

React lets you define components as functions or classes. While these two types of components differ in syntax and use different mechanisms for handling state and updates (more on that later) they can be combined and used together in React applications.


Function vs. Class Components?

Function components are the future of React. However, a lot of React applications running today still have a large portion of their React codebase and components written as classes. While the industry is steadily moving towards using only functional components and hooks, class components will still be supported in React for the foreseeable future.

At Facebook, we have tens of thousands of components written as classes, and we have absolutely no plans to rewrite them. Instead, we are starting to use hooks in the new code side by side with classes.

– Introducing Hooks, n.d., “Gradual Adoption Strategy” section. ReactJS Documentation

So the key point is that you should use functional components when creating your React apps. Still, you should also be familiar with the basics of class components, as navigating and working with the older React codebases may be part of your work in the future.


Function Components

Functional components are just JavaScript functions and they let you return/render JSX content to the DOM. We create a functional component by writing a JavaScript function:

function ExampleFuncA() {
  return (
    <div>
      <h2> ExampleFuncA - functional component </h2>
    </div>
  );
}

export default ExampleFuncA;

We can also create a functional components using the arrow function syntax:

const ExampleFuncB = () => {
  return (
    <div>
      <h2> ExampleFuncB - functional (arrow) component </h2>
    </div>
  );
};

export default ExampleFuncB;

Class Components

We create a class component by defining a class that extends React.Component.

When creating a class component we must import React in the file.

As well, class component must contain a render(){} method:

import React from 'react';

class ExampleClassA extends React.Component {
  render() {
    return (
      <div>
        <h2> ExampleClassA - class component </h2>
      </div>
    );
  }
}

export default ExampleClassA;

You might also see the React.Component class referenced as Component class. If we use this syntax, we must import and destructure component from the react module:

import React, { Component } from 'react'; // <--

class ExampleClassB extends Component {
  render() {
    return (
      <div>
        <h2> ExampleClassB - class component </h2>
      </div>
    );
  }
}

export default ExampleClassB;

In the above example, we created our class components by extending React Component. We do this to make our class inherit all the useful built-in properties and methods of a React component. Both of the previous examples are the same, just in the second example, we are using a named import.


Remember! Importing React from the react library is always required in every class component. Class components must extend React.Component which requires us to import React.

The functional component is, as its name says, a JavaScript function.

In the folder ./src/components create another file Greeting.js. This will be our new functional component Greeting:

As you can see, Greeting is just a function, with one main characteristic - it returns JSX so it can render the message on the DOM. We used the Greeting in the App function, wrapping it as a tag <Greeting />, which is how we instantiate components in React. We can reuse the Greeting component multiple times. Let’s update App.js to do so:

// App.js
// ... imports stay unchanged

function App() {
  return (
    <div className="App">
      <Navbar />
      <Greeting />
      <hr />
      <Greeting />
      <hr />
      <Greeting />
    </div>
  );
}

export default App;

As shown, it is pretty easy to reuse a component. However, as you have probably noticed, each instance of the Greeting component shows the exactly same content. In the next step, we will see how we can reuse the component and make it show custom content by passing it data through props.

Introduction - Know how the Component tree works

Components are the core building block of React apps. A typical React app is composed of many components. The structure of a React app can be represented as a component tree. The tree starts with one root component (usually named App) and has an infinite amount of nested child components.

To illustrate this, we’ll create and display a new component in our existing app. In the folder src/components create a file Button.js and add the following content to it:

// ./src/components/Button.js

function Button() {
  return (
    <a href="https://reactjs.org">
      <button> Click here </button>
    </a>
  );
}

export default Button;

Next, we will import and use the new component Button to display two buttons in the Navbar :

// ./src/components/Navbar.js
// ... previous import stays unchanged

import Button from './Button';

class Navbar extends React.Component {
  render() {
    return (
      <nav>
        <p>React - Components & props</p>
        <Button />
        <Button />
      </nav>
    );
  }
}

export default Navbar;

After rendering the component, we can represent our app in the form of a component tree that looks like this:



The component tree model allows us to hierarchically represent the relationship between components. As you can see it is similar to the DOM tree. For example, using the component tree model we can see that components Navbar and Greeting are all nested directly inside of the App.js.

Thinking about a React app structure in this way gives us a bird’s-eye view of the app. This is needed to allow us to visualize the data flow when passing data between the components. We will see how to pass data between the components by using props in the next step.

Props

One of the fundamental concepts when dealing with React components is props. Props represent the data that component receives from the outside.

Conceptually, components are like JavaScript functions. They accept arbitrary inputs (called props) and return React elements describing what should appear on the screen.

Passing and accessing a prop

The term props stands for properties. In JSX, when a prop is sent to a component, it looks like an HTML attribute. Let’s take a look. In the App function, add the following piece of code:

// App.js
// ... imports stay unchanged

function App() {
  return (
    <div className="App">
      <Navbar />
      <Greeting firstName="Harper" />
      <hr />
      <Greeting />
      <hr />
      <Greeting />
      <hr />
    </div>
  );
}

The custom attribute firstName="Harper" that we added to the component is a prop. The prop firstName and its value "Harper" will be available inside the component through the props object.

To access the props object we first have to specify it as the parameter of the function component:

function Example(props) {
  return /* JSX content */;
}

Let’s update the Greeting component to see this in practice. Add the parameter props and console.log it in the console:

// ./src/components/Greeting.js
function Greeting(props) {
  const message = 'Hello Ana!';

  console.log(props);

  return (
    <div>
      <u>{message}</u>
    </div>
  );
}

export default Greeting;

The keyword props is an object. It holds the values that we pass to the component as props, from the outside. In this case, props has a property firstName with the value "Harper". Open the browser’s console and check the output:



We can access the property firstName and display it in the component. Update the Greeting component :

// ./src/components/Greeting.js

function Greeting(props) {
  // const message = 'Hello Ana!';   <== REMOVE
  console.log(props);

  return (
    <div>
      <u>Hello {props.firstName}!</u>
    </div>
  );
}

export default Greeting;

Knowing that we can pass any data to a component using props, opens up so many opportunities for component reusability. Let’s update the <Greeting> components and pass a firstName props to each:

// App.js
// ... imports stay unchanged

function App() {
  return (
    <div className="App">
      <Navbar />
      <Greeting firstName="Harper" />
      <hr />
      <Greeting firstName="Michelle" />
      <hr />
      <Greeting firstName="Andrea" />
      <hr />
    </div>
  );
}

// ...

Props are pieces of information that come from outside of the component. Another way to describe them is they are the data passed into the component.


Passing multiple props

You can pass as many props as needed to a component. Also, the prop value passed can be of any data type (string, number, boolean, array, object, function). We’ll use a new component StudentCard to see how to pass props that have different data types:

  1. Let’s create a new file ./src/components/StudentCard.js and the component StudentCard:
// ./src/components/StudentCard.js
import React from 'react';

function StudentCard(props) {
  return (
    <div className="StudentCard">
      <p>
        <b>Name:</b> NAME
      </p>
      <p>
        <b>Cohort:</b> CITY - COURSE
      </p>
      <p>
        <b>Current week: WEEK </b>
      </p>
      <br />
    </div>
  );
}

export default StudentCard;
  1. Next, we’ll import the component StudentCard and display it inside of App.js. This time we’ll pass multiple props: name - string, week - number and info - object:
// App.js
// ... imports stay unchanged

import StudentCard from './components/StudentCard';

function App() {
  return (
    <div className="App">
      <Navbar />
      <Greeting firstName="Harper" />
      <Greeting firstName="Michelle" />
      <Greeting firstName="Andrea" />

      <StudentCard name="Eva" week={7} info={{ city: 'BCN', course: 'WEB' }} />
      <StudentCard name="Mat" week={8} info={{ city: 'MIA', course: 'DATA' }} />
    </div>
  );
}

export default App;
  1. StudentCard component receives props name, week, and info from App. We will update StudentCard.js to display them:
// ./src/components/StudentCard.js
import React from 'react';

function StudentCard(props) {
  return (
    <div className="StudentCard">
      <p>
        <b>Name:</b> {props.name}
      </p>
      <p>
        <b>Cohort:</b> {props.info.city} - {props.info.course}
      </p>
      <p>
        <b>Current week: {props.week}</b>
      </p>
      <br />
    </div>
  );
}

export default StudentCard;

Take a note of the object syntax we are using to access the data coming from the info object: props.info.city and props.info.course.


props in the class component

We saw that accessing the props object is pretty straightforward when it comes to functional components. Let’s create a new component StudentList as an example of how to use props in class components:

// ./src/components/Badge.js
import React from 'react';

class Badge extends React.Component {
  render() {
    return <span className="Badge"> {this.props.badgeText} </span>;
  }
}

export default Badge;

Next, we’ll import the component and display it inside of App.js and pass it the prop badgeText="important":

// App.js
// ... imports stay unchanged

import Badge from './components/Badge';

function App() {
  return (
    <div className="App">
      <Navbar />
      <Greeting firstName="Harper" />
      <Greeting firstName="Michelle" />
      <Greeting firstName="Andrea" />

      <StudentCard name="Eva" week={7} info={{ city: 'BCN', course: 'WEB' }} />
      <StudentCard name="Mat" week={8} info={{ city: 'MIA', course: 'DATA' }} />

      <Badge badgeText="important" />
    </div>
  );
}

export default App;

If we are inside a class, we have to use the this keyword to access the props. This is why we used {this.props.badgeText} inside of the class component Badge.

Composing components - props.children

So far we saw that components are mostly self-closing tags, such as <User />.However, you can also write components as if they were regular tags:

<CoolComponent> </CoolComponent>

This allows us to use a component as a wrapper for one or more other components. Let’s create a new component StudentList to show this:

// ./src/components/StudentList.js

function StudentList(props) {
  return (
    <div>
      <h2>Student List</h2>
      {props.children}
    </div>
  );
}

export default StudentList;

By creating the component in such a way we say: “this component will be a “parent” component or “wrapper” component for other components”.

To show this, Let’s wrap all the StudentCard components with the UserList component:

// App.js
// ... previous imports stay unchanged

import StudentList from './components/StudentList';

function App() {
  return (
    <div className="App">
      <Navbar />
      <Greeting firstName="Harper" />
      <Greeting firstName="Michelle" />
      <Greeting firstName="Andrea" />
      <Badge badgeText="important" />

      <StudentList>
        <StudentCard name="Eva" week={7} info={{ city: 'BCN', course: 'WEB' }} />
        <StudentCard name="Mat" week={8} info={{ city: 'MIA', course: 'DATA' }} />
      </StudentList>
    </div>
  );
}

As we can see, nothing major changed in the way how the content is rendered.

Having the opportunity to use props.children allows us to compose components. We will be taking advantage of this feature a lot.


So far, we have covered props and we now know that props allow us to pass data in the component. Explained with industry terms, we could say that props allow you to pass data down the component tree.

One thing to keep in mind: props are immutable.

When do you use props?

(1) Props are like function arguments

You can use props to make your components reusable. Props can be used to control how the component is displayed. Passing in different props often results in the component looking or behaving differently.

In this way, props for a component are like arguments of a function. Arguments control how a function behaves, just like props control how the component is displayed.

Imagine we had a cool component to display the weather of a particular city. In that case, the prop we send would probably be the city name. Using that name, it would probably access the weather data from an API.

Rendering the component would probably look like this:

// This code is just hypothetical
// ...
  render() {
    return <WeatherWidget city="Mexico City" />;
  }
// ...

Sending a different city prop should produce different information.



(2) Share data between components

By definition, props let you share data between a parent component and a child component. Sharing data this way is very common. It’s quite common for an action that happens in one component to affect how another component should behave.


(3) Components from npm

There are many pre-made components on the npm. Often, components you get from the npm will allow you to configure and customize them through the use of props. Let’s see an example!

First, let’s install the react-player npm package.

$ npm install react-player

Now let’s import it inside App.js and use it as any other component:

// App.js
// ... previous imports stay unchanged

// import ReactPlayer from the npm package
import ReactPlayer from 'react-player';

function App() {
  return (
    <div className="App">
      {/* ... no changes here ... */}
      <hr />
      <ReactPlayer url="https://vimeo.com/channels/top/22439234" playing />
    </div>
  );
}

export default App;

As the documentation states, the basic usage only involves the url prop – the address of the video we want to display. It works with many of the popular video services like YouTube and Vimeo. Aside from url, the component also has other props that you can read about in the package’s documentation. We also used the playing prop – a boolean prop that controls auto-play.

Additionally, if you don’t like the most popular YouTube video of the day? Maybe two new props can help! The controls prop adds player controls and the volume prop lets you control the volume.

// App.js
// ... previous imports stay unchanged

  render() {
    return (
      <div className="App">
      {/* ... no changes here ... */}

        <ReactPlayer url="https://vimeo.com/channels/top/22439234" />

        {/* see the new props! */}

        <ReactPlayer
          url="https://www.youtube.com/watch?v=kJQP7kiw5Fk"
          playing
          controls
          volume="0.5"
        />
      </div>
    );
  }
// ...

With a component like ReactPlayer, you can see how props allow you to customize the way the component works for you.
