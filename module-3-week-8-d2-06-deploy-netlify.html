Learning goals

After this lesson, you will be able to:

  • Deploy your frontend application to Netlify.

Netlify

Netlify is an all-in-one platform for automating modern web projects. Netlify supports all modern single-page application (SPA) frameworks: React, Angular, Vue, but you can deploy just a simple HTML/CSS website using Netlify as well.

The image below is taken from the official Netlify website and summarizes the core of the deployment process.



Netlify is a really powerful platform. It supports the whole hosting infrastructure, continuous integration, and deployment pipeline with a single workflow.

We will take you through the simple process of deploying your front-end application on Netlify, and we leave it up to you to explore all the extra features later on.


Getting started

Although we will take you through the step-by-step process, it is important to understand that, from a higher perspective, deploying to Netlify is as simple as connecting your existing GitHub repo with Netlify. Here we come to the continuous deployment support - which basically means that all you have to do is push your code and Netlify will do the rest, publishing your website for you.


Step 1 | Create an account on Netlify


  1. Head to Netlify.

  2. Click on the signup button located in the upper right corner.

    Check the visual.

  1. Sign up using GitHub by clicking on GitHub (if you do not have a GitHub account stop here and make a GitHub account first).

    Check the visual.

  1. If you don’t have the autocomplete turned on, you will be prompted to enter your GitHub credentials.

    Check the visual.

  1. It is essential that you click on “Authorize Netlify for GitHub”.

    Check the visual.
  1. You will be presented with 3 steps and different screens. Click “Next” on the first 2 and “Finished” on the last one.

You are all set. You can proceed to deploy your first app on Netlify.


Step 2 | Deployment


  1. After you create an account and finished all the steps mentioned above, you will be taken to https://app.netlify.com. On this page, you can choose to create a website without connecting to Git with simple drag and drop or to create a website connecting it with GitHub repo. We will go for the second option since it will allow us to enable continuous deployment - every change we make locally and push to GitHub will be “picked” up by Netlify.

    So, when here, choose New site from Git.

    Check the visual.

  1. Click on GitHub under Continuous Deployment.

    Check the visual.

  1. A new window will open, and you might be asked to authorize Netlify once more. Click on Install and make sure to leave All repositories checked. If you don’t see this window, you will just see the message - Authorized.

    Check the visual.

  1. The window will close and you will now be on step 2. Choose the repository you want to deploy.

    Check the visual.

  1. You will now be on step 3. Finish deploying by clicking on the Deploy site.

    Check the visual.

  1. You will be taken to the Overview page and Netlify will take a couple of minutes to finish deploying. The Site deploy in progress notification should be yellow. Once Netlify finishes you will see that the notification is gone and replaced by the link that turned from yellow to green. This means your website is now live and you can see the URL of your newly published website. And that is it! You have your app deployed!

    Netlify automatically generates a name for your site. You can update it by visiting the Settings tab and then under General clicking “Change site name”:

    Check the visual.

Step 3 | Setting up the environment variables


Go to the Heroku app and in the Settings tab, go under Reveal Config Vars and add the URL to your newly created Netlify app. Make sure you add it under the same name as you used it in your server-side app code. For example - if you set the FRONTEND_URL in the CORS, then use the same name in the Heroku app:

// app.js

// Cross-Origin Resource Sharing

app.use(
  cors({
    origin: process.env.FRONTEND_URL,
    credentials: true
  })
);

Summary

In this learning unit, we covered the deployment process of a React front-end application with Netlify. After successfully deploying the application with Netlify, we updated our server’s CORS configuration to allow network requests coming from our front-end application.
